Getting Started

Downloading

First of all you will need to log in to your Themeforest account.

After that goto your downloads page

From there locate the Verso template click the download button and then select All files & documentation

Congratulations. You now have a shiny new copy of Verso

Unzipping

You should now have a file that looks something like this.

verso-1.0.0.zip

In order to get at the goodness inside you will need to unzip this file.

Zip Contents

Once you have unzipped the Verso zip file you will find the following folders have been created.

Folder / File Description
docs/ The docs your are looking at now
html/ Copy of the demo site
source/ Copy of the template source with Gulp development server
psds/ Contains the templates PSD files

HTML or Gulp

This template was built using the Nunjucks templates which are compiled into HTML using Gulp.

We provide both the plain HTML files and the full Nunjucks source code with this template.

So you can either dive in and edit the HTML files that are inside the /html folder or install Gulp and get started using the Nunjucks templates which are in the /source folder.

HTML

File Structure

When you open an html demo folder from the .zip file that you downloaded you will find the following file structure

Folder / File Description
demo-folder/ Each demo comes in its own folder, i.e corporate-classic
demo-folder/assets/ Contains all the necessary styling and logic for your website
demo-folder/assets/css/ Contains the CSS that the theme uses
demo-folder/assets/fonts/ Contains the fontawesome font files
demo-folder/assets/images/ Contains all the images of the theme
demo-folder/assets/js/ Contains all javascript files that the theme needs
demo-folder/vendor/phpmailer/ Used by the contact forms in the theme to send the form email
demo-folder/*.html The html files that render the theme pages for each demo

                    </div>
                </div>
            </section>
                .
                . (more sections)
                .
            <footer>
                Add your footer widgets
                Add your subfooter
            </footer>

    </html>

Deploying

Deploying your website on a server requires that you have the necessary ftp credentials provided by your hosting provider. All you need to do is copy your files over to your server using an ftp client such as Filezilla. Pick the demo that you want to start with and copy all the files from the respective demo folder.

Gulp Setup

The source files

Once you have downloaded and unzipped the main template download zip, you will find a folder called /source. This folder contains all the source code you will need to build the template.

The template uses gulp to build the source files into HTML / CSS and JS files that you can then deploy to your server.

The build system also uses bower to manage loading all the templates' dependencies.

Prerequisites

Before you start to set up the template development server you will need to install nodejs and bower.

Installing Node.js

We recommend installing the latest version of node.

You can download the latest of node.js from here alternatively there are guides on how to install using a package manager.

Checking node

Once you have node.js installed on your system you should be able to go to your command line and type

node --version

and see something like this

v4.1.2

Installing bower

If you do not have bower installed you should go to your command line and type

npm -g install bower
Note - You will need git installed for bower to work. Visit Git website

Checking bower

Once you have node.js installed on your system you should be able to go to your command line and type

bower --version

you should see something like this

1.5.2

Setup Development Server

Use the source

To start off you will need to open the /source folder contained in the download.

Installing node packages

Open a command prompt and change directory to the /source that was created when you unzipped the source folder and run this command

npm install
Note - This may take some time to install all the packages.

Installing Git

You will also need to install the Git source control system on your machine. Check out this guide on how to install on Windows, Mac and Linux.

Installing bower packages

Using the command prompt and from the same /source directory that you installed the node packages, run this command

bower install

This will install all the components that the template needs.

Note - You will need git installed for bower to work.

Running the development server

First make sure you have gulp-cli installed globally

npm -g install gulp-cli

Verso comes shipped with different demos that you can choose from.

The format of the command to start a web server with a specific demo is

gulp serve:folder-name-of-demo

So for example to start working on a desired demo, named corporate-classic type

gulp serve:corporate-classic

This should automatically open up a browser window with the demo running.

Check the available demos here

Browsersync

Once the development server is up and running any changes you make to the HTML & JS & SASS files will be auto updated in your browser window.

Production build

Running a production build

Once you are happy with your site you can initiate a build that will create a copy of the template that you can FTP to your web server.

Verso comes shipped with different demos that you can choose from. To initiate a build with a demo named i.e corporate-classic, run the command

gulp build:corporate-classic

This will initiate a build, once it has finished you will find your built files in a folder called dist that will have been created for you.

<a href="index.html">

Change the src attribute of that image to point to your logo.

From the code used in the previous section

<a href="index.html">

Change the href="index.html" to point to the page that you wish should set the logo link url.

Fonts

Default fonts

The default fonts that Verso is using is Quattrocento(Corporate Classic), with the styles Normal 400, Bold 700. Also, Verso is using Oswald, with the styles Light 300, Normal 400, Bold 600, Bold 700. The fonts are set inside <head> using the link that Google Fonts provide.

It should look like

<link href="https://fonts.googleapis.com/css?family=Quattrocento:400,700%7COswald:300,400,500,600,700" rel="stylesheet">

Changing the site fonts

If you want to change the site fonts, you need to load them first. To do that, should you want to use Google Fonts, you need to visit the Google Fonts page and create the import link that you will use inside the <head>.

If you are new to Google Fonts, follow these simple steps to create the new import link:

  1. Choose: Search or browse hundreds of font families, then add the ones you like to your Collection.
  2. Review: Compare and refine your Collection, even see the styles in a dynamic sample layout.
  3. Use: Grab the code prepared and you’re ready to add the Collection to your website!

Once you have the new link, replace the default one with the new one in the <head>.

In order for the new font to be applied to the body of Verso, make sure that you edit source/html/src/demo-folder/assets/scss/variables.scss and set line 6 that looks like

    $font-family-base:  'Quattrocento';

to use the font that you want. Now your font is applied to the body. To change the font for the headings check line 12 that looks like

    $headings-font-family: 'Oswald';

Typography

Headings

Bordered Heading

To add a border on the left of a heading, use the verso-heading-bordered class. For example

<h2 class="verso-heading-bordered text-capitalize">
    <small class="text-muted font-italic">About</small> Verso
</h2>

Bordered Image Heading

Instead of a border, a heading can use an image. For example

<h2 class="verso-heading-image verso-heading-image-01 verso-heading-image-reversed">
    Save
    <span class="d-block">The date</span>
</h2>

Reversed Bordered Heading

A bordered heading can be reversed, placing the border on the right of the title. Adding the verso-heading-bordered-reversed should do it, code example

<h4 class="verso-heading-bordered verso-heading-bordered-reversed text-capitalize">
    <a href="video-post.html">
        <small class="text-muted font-italic">Next →</small> The Beast &amp; the Book
    </a>
</h4>

Font Weight

To add a special font weight to your text you need to apply the verso-font-weight-n class, where n=100, 200, ..., 900, depending on the font weights that your font loads, remember the font import at the Fonts section. For example

<span class="pull-right verso-font-weight-700">$2000 total</span>

Line height

To change the line-height property of an element you can use the following classes:

  • verso-line-height-small: sets the line height to 1
  • verso-line-height-normal: sets the line height to whatever value is set in the skin. Check the skin variable $line-height-base inside source/html/src/demo-folder/assets/scss/variables.scss
  • verso-line-height-big: sets the line height to twice the size of verso-line-height-normal

Light Text Color

You can set an alternative light color for your text that changes the default text colors, in case you are using background colors on elements that do not match the default text colors. For example, when you have a black background color and your text color is dark as well, using a light text will make the text stand out. Check out the variables inside source/html/src/demo-folder/assets/scss/variables.scss, lines 130-135

// Light colored elements
$o-light-color: #f8f2e6;
$o-light-headings-color: #f8f2e6;
$o-light-link-color: #f8f2e6;
$o-light-link-hover-color: #f8f2e6;
$o-light-text-muted: #f8f2e6;
$o-light-border-color: #f8f2e6;

To add the light color style to your elements use the verso-text-breakpoint-light class by replacing the breakpoint notation to either blank, as in verso-text-light for Extra Small or sm, md, lg or xl for Small, Medium, Large or Extra Large container widths.

Footer

Footer structure

The footer of Verso has the same layout as a section. The code for the footer is

    <footer class="verso-footer">
        <section class="section verso-py-9">
            <div class="section-background">
                <div class="section-bg-image section-bg-image-size-cover section-bg-image-position-bottom verso-demo-bg-image-09 section-bg-image-paralax verso-opacity-2"></div>
            </div>
            <div class="section-content">
                <div class="container">
                    <div class="row">
                      Place your widgets here...
                    </div>
                </div>
            </div>
        </section>
    </footer>

For each of the widgets that are placed inside the footer, for a 3-column footer layout use .col-md-4, as in

    <div class="col-md-4">
        <div class="verso-widget widget_text">
            <h4 class="verso-widget-header">About Us</h4>
            <p>Why do the nations rage, and the peoples plot a vain thing? </p>
            <p>The kings of the earth take a stand, and the rulers take counsel together.</p>
        </div>
    </div>

To use the already styled widgets for Verso, use the code provided in source/html/partials/widgets/ folder. There you will find the HTML used to render the following widgets: Categories, Events, Menu, Recent Posts, Search, Social, Tags and Text.

Adding a background image to the footer

To add a background image to the footer you can assign a class to the section-background div and use CSS to add the image as background. As shown in the example above, the verso-demo-bg-image-09 class is added and the following CSS sets the background-image property

.verso-demo-bg-image-09 {
    background-image: url(../images/church-09-1600x1000-notinclude.jpg);
}

Adding a widget to the footer

What we need to add a widget is firstly a title. Use this markup to add a title as an <h4>

    <h4 class="verso-widget-header">About Us</h4>

Everything that is considered as the content of the widget can either be a list or not. If it is a list, like a list of products for example, we need to wrap it inside

    <ul>
        <li>...</li>
        <li>...</li>
        <li>...</li>
    </ul>

If it is not a list, we need to wrap it inside

    <p>
        content
    </p>

We can add there as a content anything we like, images, text, icons etc.

Adding a sub footer

You can add a mini footer if you use markup similar to the footer.

    <section class="section">
        <div class="section-content">
            <div class="container">
                <div class="row">
                    <div class="col-sm-6 text-sm-left">
                        Add your widget code
                    </div>
                    <div class="col-sm-6 text-sm-right">
                        Add your widget code
                    </div>
                </div>
            </div>
        </div>
    </section>

This will render a subfooter section split into two columns, each holding one widget. Use that code before closing the main footer to add a subfooter at the end of your pages.

Layout

Boxed Layout

To make your website use a boxed layout you need to do the following:

  1. Edit your variables.scss file, it should be located in source/html/src/demo-folder/assets/scss/variables.scss.
  2. Locate the $o-box-layout-width variable around line 401. Change that to something less than 100% and that will be the width that your content will have, producing a boxed layout effect.
  3. Set a background color for the page outside your main content by setting the $o-box-layout-bg on the same file.

Page Layout

The layout of the pages on Verso is defined in default.html inside the source/html/src/demo-folder/layouts folder. If you take a look at this file, the main components are

  • <head>: is consisted of some meta data and the imports for the fonts and css.

  • <body>: this is the main content of the page. First, we define the header. Your header gets imported from the source/html/src/demo-folder/layouts/headers folder

      <div class="verso-content-box verso-content-box-move-behind">
      {% block header %}
          {% include "partials/headers/default.html" %}
      {% endblock %}

Now is the time to add your pages' content

        <div class="verso-content">
            {% block content %}{% endblock %}
        </div>

Add your footer

        {% block footer %}
            {% include "partials/footer.html" %}
        {% endblock %}
    </div>

To end with, before you close the <body>, you need to import your scripts.

    <script src="assets/js/theme.js" charset="utf-8"></script>
    {% block extrajs %}{% endblock %}

Page Sections

In order to create a section, just create a div and give it a class section. You can insert grids inside sections in order to populate them with content. The content has to be wrapped by a section-content container, as in the example below:

    <div class="section">
        <div class="section-content">
            <div class="container">
                <div class="row">
                    <h1>A header inside a section!</h1>
                </div>
            </div>
        </div>
    </div>

Besides using grids to insert content into sections as we saw above, you can customize your sections and give them unique colors and background images.

Create a section with a color background

We will build on the example code that we previously used, and we will add some coloring. To add a background color to a section, you need to add a class that defines a background color to the outer section wrapper. This class should exist inside a file named custom-css.css inside assets/css/ folder for the built version or /source/html/src/demo-folder/assets/scss/ folder for the source version. Feel free to create your own classes, such as

.verso-demo-bg-1 {
  background: #e8e2d7;
}

The code that renders the section with a background color is

<div class="section verso-demo-bg-1">
    <div class="section-content">
        <div class="container">
            <div class="row">
                <div class="col-md-6">
                    <h1>Section Title</h1>
                    <p class="lead verso-line-height-normal">
                        Section subtitle
                    </p>
                </div>
            </div>
        </div>
    </div>
</div>

Create a section with an image background

Sections can have background images. Add a div with an section-background class before your content. Then, inside that div add another div with the class section-bg-image and an extra class such as verso-demo-bg-image-1 that loads the class which will load the background image(to avoid inline styles). The class that loads that image should be inside a file named custom-css.css inside assets/css/ folder for the built version or /source/assets/scss/ folder for the source version. Feel free to create your own classes, such as

.verso-demo-bg-image-1 {
  background-image: url('../images/verso/verso-01-1600x1000.jpg');
}

The code that renders the section with a background image is

<div class="section">
    <div class="section-background">
        <div class="section-bg-image verso-demo-bg-image-1"></div>
    </div>
    <div class="section-content">
        <div class="container">
            <div class="row">
                your content
            </div>
        </div>
    </div>
</div>

Section background image position

You can configure the alignment of the background image using one of the following classes: section-bg-image-position-top, section-bg-image-position-middle and section-bg-image-position-bottom. Add one of those classes to the section-bg-image div that holds your background image. For example the following code will align the background image at the bottom:

<div class="section">
    <div class="section-background">
        <div class="section-bg-image section-bg-image-position-bottom verso-demo-bg-image-1"></div>
    </div>
    <div class="section-content">
        <div class="container">
            <div class="row">
                your content
            </div>
        </div>

Utilities

Margin

Adding margin to an element is a matter of using the appropriate classes. The following classes are provided:

  • verso-mt--30 : Adds 300px of negative margin to the top of an element.
    This class has variations that grow up to zero margin and continue up to verso-mt-30, which adds 300px. Classes scale by 10px for each index increase. You can use verso-mt-29, verso-mt-28, ..., verso-mt-0, verso-mt--1, verso-mt--2, ..., verso-mt--30.
  • verso-ml--30 : Adds 300px of negative margin to the left of an element. Variations follow the same pattern as above.
  • verso-mr--30 : Adds 300px of negative margin to the right of an element. Variations follow the same pattern as above.
  • verso-mb--30 : Adds 300px of negative margin to the bottom of an element. Variations follow the same pattern as above.
Note that verso-mt-30 adds margin whilst verso-mt--30 adds negative margin.

Margins For All Devices

To add even more control over the top/bottom margins on different screen sizes, we have introduced a set of rules that define different margins on different devices.
The classes are set as verso-mt-breakpoint-n or verso-mb-breakpoint-n where n = -30, -29, ..., 29, 30(each number removes/adds 10px) and replacing the breakpoint notation to either sm, md, lg or xl for Small, Medium, Large or Extra Large container widths. If breakpoint is set to blank, as in verso-mt-30 for example, it affects Extra Small devices and up. See Bootstrap 4 grid system.

Padding

Adding padding to an element is a matter of using the appropriate classes. The following classes are provided:

  • verso-py-1 : Adds 10px of padding to the top and bottom of an element.
    This class has variations that scale up to 300px padding. Classes scale by 10px for each index increase. You can use verso-py-1, verso-py-2, ..., verso-py-30.
  • verso-px-1 : Adds 10px of padding to the left and right of an element. Variations follow the same pattern as above.
  • verso-pt-30 : Adds 300px of padding to the top of an element. Variations follow the same pattern as above.
  • verso-pl-30 : Adds 300px of padding to the left of an element. Variations follow the same pattern as above.
  • verso-pr-30 : Adds 300px of padding to the right of an element. Variations follow the same pattern as above.
  • verso-pb-30 : Adds 300px of padding to the bottom of an element. Variations follow the same pattern as above.
  • verso-pa-30 : Adds 300px of padding around an element, top, bottom, left and right. Variations follow the same pattern as above.

Paddings For All Devices

To add even more control over the top/bottom/left/right paddings on different screen sizes, we have introduced a set of rules that define different padding on different devices.
The classes are set as verso-pt-breakpoint-n or verso-pb-breakpoint-n where n = 0, 1, ..., 29, 30(each number adds 10px) and replacing the breakpoint notation to either sm, md, lg or xl for Small, Medium, Large or Extra Large container widths. If breakpoint is set to blank, as in verso-pt-30 for example, it affects Extra Small devices and up. See Bootstrap 4 grid system.

On scroll animations

To add an animation to an element, you need to assign the verso-os-animation class to it. In addition, you need to define the type of animation and when it is going to kick in. These are set from the two data attributes data-os-animation and data-os-animation-delay. An example

<h1 class="verso-os-animation" data-os-animation="fadeIn" data-os-animation-delay=".3s">
    <small>Dont miss</small> Upcoming Event
</h1>
To view the full set of the available animations visit Animate.css
Set the data-os-animation-delay attribute in seconds, i.e data-os-animation-delay="0.5s", for half a second. This means that the animation takes place half a second after the related item shows on the page.

Opacity

Setting the opacity level for an element in Verso is a matter of using the appropriate classes. The following classes are provided:

  • verso-opacity-0 : Completely transparent.
    This class has variations that scale up to 10. Classes scale by 1 each. You can use verso-opacity-1, verso-opacity-2, verso-opacity-3, ..., verso-opacity-10.
  • verso-opacity-10 : Not transparent.

Shadow

Setting the shadow level for an element in Verso is a matter of using the appropriate classes. The following classes are provided:

  • verso-shadow-0 : No shadow.
    This class has variations that scale up to 20. Classes scale by 1 each. You can use verso-shadow-1, verso-shadow-2, verso-shadow-3, ..., verso-shadow-20.
  • verso-shadow-20 : Maximum shadow effect.

Shadow applied when the user hovers over an element.

  • verso-shadow-hover-0 : No shadow.
    This class has variations that scale up to 20. Classes scale by 1 each. You can use verso-shadow-hover-1, verso-shadow-hover-2, verso-shadow-hover-3, ..., verso-shadow-hover-20.
  • verso-shadow-hover-20 : Maximum shadow effect.

Shadow applied when an element is focused.

  • verso-shadow-focus-0 : No shadow.
    This class has variations that scale up to 20. Classes scale by 1 each. You can use verso-shadow-focus-1, verso-shadow-focus-2, verso-shadow-focus-3, ..., verso-shadow-focus-20.
  • verso-shadow-focus-20 : Maximum shadow effect.

To setup the z-index of your element, you can use the following classes:

  • verso-z-0 : s-index is 0.
    This class has variations that scale up to 10. Classes scale by 1 each. You can use verso-z-1, verso-z-2, verso-z-3, ..., verso-z-10.

Positioning

To give an element a relative or absolute position you can use the following classes:

  • verso-pos-relative : the element is positioned relative to its normal position.
  • verso-pos-absolute : the element is positioned relative to the nearest positioned ancestor.

Note: A "positioned" element is one whose position is anything except static.

Changing the map options

In order to change the location of the map you need to edit source/html/contact.html. Inside that file locate the code for the map options

    <script>
        window.map = {
            map_type: 'ROADMAP',
            auto_zoom: 'manual',
            map_zoom: 7,
            map_scrollable: 'on',
            map_style: 'retro',
            addresses: ['London', 'Paris',],
            latlngs : ['51.511084, -0.133202','51.506623, -0.111916'],
            labels: ['London', 'Paris'],
            auto_center: 'auto',
            center_latlng: '',
            show_markers: 'on',
            markerURL: 'assets/images/marker.png',
        };
    </script>

The available options are

  • map_type : ROADMAP, SATELLITE, TERRAIN or HYBRID.
  • auto_zoom : set to 'manual' to set the zoom manually. Leave empty for auto zoom
  • map_zoom : the amount of zoom that you wish to have (1-20), if auto_zoom is set to 'manual'
  • map_style : blackwhite, retro or regular
  • map_scrollable : on or off for the map to be draggable
  • show_markers : on or off.
  • labels : Set the text that will show as the markers label. If multiple markers are added, make sure that you have set the same number of labels as well.
  • addresses : Set the addresses of the map's markers. Be careful of the syntax that is being used here. It has to be an array of comma-separated single quotes. Inside the quotes put the addresses separated by comma. Leave empty if you want to use lat/lng. Example for 3 addresses ['address 15 New York','address 22 New Orleans','address 3 Amsterdam']
  • latlngs : Set the latitude and longitude of the map's location. Be careful of the syntax that is being used here. It has to be an array of comma-separated single quotes. Inside the quotes put the latitude and longitude separated by comma. Example for 3 markers ['lat1, lng1','lat2, lng2','lat3, lng3']
  • auto_center : Set to auto if you want the map to be centered automatically, fitting all the markers that you have inserted or custom if you want to set the center of the map
  • center_latlng : Set the lat/lng according to which you want the map to be centered at, unless auto_center is set to auto
  • markerURL : Set the URL of the marker's icon.

Advanced

PHP Contact Form

The contact form uses the PHPMailer, which is a full-featured email creation and transfer class for PHP. For more about PHPMailer please visit the official website http://phpmailer.worxware.com/.

Connecting the Contact Page to PHPMailer

For your contact page to connect to PHPMailer, you need to make sure that your form has an action set to call contact-mailer.php on the server, as in

    <form action="contact-mailer.php" class="contact-form" id="contactForm">

Also, you need to make sure that there is a folder named vendor in the root of the theme. This is the PHPMailer plugin. For the ajax call to take place, it is also vital that you assign the contact-form class to the form, which triggers the related javascript.

For the contact form to work properly, you need to add to the related page

    <script type="text/javascript" src="assets/js/contact.js"></script>

Setup PHPMailer

To setup your contact form you will need to edit the contact-mailer.php file in the main theme folder. The first 62 lines can be modified to customise your contact form.

  1. Set the main host. It is by default set to smtp.gmail.com if you are using a GMail account. Other options are

    Hotmail  => `smtp.live.com`
Yahoo    => `smtp.mail.yahoo.com`
Lycos    => `smtp.mail.lycos.com`
AOL      => `smtp.aol.com`
  2. Set the SMTP username and password. This is the email account that PHPMailer will use to send the email. Change some_email@email.com to the email that you want to use and secret to the password of that email, from lines 16, 17.
  3. Set the address that the email of the form will be delivered to. Change the receiver@mail.com from line 21 to the address that you want.
  4. Set the company name - When a message arrives in your inbox it will have a subject with your company name inside. To change this edit line 26 and change the your_company to your company name.

  5. Set which fields are required - to change which fields are required in your form you can edit the following

     $field_rules = array(
  'name' => 'required',
  'email' => 'required|valid_email',
  'message' => 'required'
 );

    Remove the lines of the fields you do not want to be required.

  6. Change error validation messages - to change the error messages that popup on the forms change the following section

     $error_messages = array(
  'required' => 'This field is required',
  'valid_email' => 'Please enter a valid email address'
 );
 // select where each inputs error messages will be shown
 $error_placements = array(
  'name' => 'top',
  'email' => 'top',
  'subject' => 'right',
  'message' => 'right',
  'submitButton' => 'right'
 );

$error_messages - defines the messages shown in the popups.

$error_placements - defines where the popups will appear on (top|bottom|left|right).

Keep in mind that you need to include the <script src="assets/js/contact.js"></script> at the end of the <body>.

Troubleshooting

If you have done all of the above as instructed and you still have issues with the contact form, please do the following:

  1. Contact your host and ask them if PHP is enabled and if outgoing connections are enabled.
  2. Check if the vendor folder exists on your server.

  3. Enable debug mode on PHPMailer and get more info on the problem. You can enable debug mode if you add the following line to the beginning of contact-mailer.php, after

     $mail = new PHPMailer;
 $mail->SMTPDebug  = 2; // enables SMTP debug information (for testing)
                // 1 = errors and messages
                // 2 = messages only
  4. Follow the instructions provided by PHPMailer troubleshooter here.
